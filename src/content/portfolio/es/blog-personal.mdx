Construí este blog para tener un espacio propio donde escribir y mostrar mi trabajo. La arquitectura es interesante: i18n nativo con slugs distintos por idioma vinculados por un `translationKey` en el frontmatter, view transitions tipo morph entre la lista y el detalle de cada post, y un endpoint serverless para el formulario de contacto.

## Decisiones técnicas clave

- **Astro + MDX:** contenido en el repo, versionado con git, cero CMS externo.
- **i18n con prefijo en ambos idiomas:** ningún idioma es ciudadano de segunda.
- **View transitions nativas:** sin JS de framework, solo la API del navegador + Astro's `ClientRouter`.
- **Starwind UI:** componentes accesibles sin runtime React.
