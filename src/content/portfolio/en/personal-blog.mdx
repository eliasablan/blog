I built this blog so I'd have a space of my own to write and showcase my work. The architecture is the interesting part: native i18n with per-language slugs linked by a `translationKey` in frontmatter, morph-style view transitions between the post list and detail page, and a serverless endpoint behind the contact form.

## Key technical decisions

- **Astro + MDX:** content lives in the repo, versioned with git, zero external CMS.
- **i18n with prefix in both languages:** no language is a second-class citizen.
- **Native view transitions:** no framework JS — just the browser API and Astro's `ClientRouter`.
- **Starwind UI:** accessible components with no React runtime.
